Pokemon Application

An application that shows a list of all the pokemon characters and their pictures with a nice looking UI. The pokemon characters were fetched from the API provided on https://pokeapi.co/ build. When a pokemon character is clicked it leads to a page with full details of the character pulled from the appropriate endpoint with all images, move, abilities, stats etc showing at different sections. When there is internet connection issues, the application reacts immediately without crashing. The external libraries used are: Retrofit Library, Picasso, RXJava2, JUnit, View binding, and Espresso. It requires permission to access the internet. The amount of information one wishes to see can be set on the application. The architecture used in building the application is MVVM architecture.

Map Tracker

A map application that shows the exact position of another person in realtime. It shows the partner's position as an indicator, the indicator will move when the partner moves. The google map API was used to achieve the task. Also, the Firebase was used to store the partner's position, it keeps updating at every two minutes interval. The architecture used to build this application is the MVC architecture. To access the users location, it makes use of the Permission system, whereby it asks for permission from the user and the reason for using it.

Tru-Contacts Application

A contact application with pages that shows a list of my saved contact. It allows new contacts to be stored on it, also has a contact details page where editing, calling, sharing and deleting of contacts can be done. The data store used is the firebase. Calling requires permission before it can be made. When a contact is clicked on, it moves to a new page through a sliding Animation, where it shows all the details of the contacts.
